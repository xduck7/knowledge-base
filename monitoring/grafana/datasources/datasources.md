# Источники данных Grafana: как это работает

Grafana сама не хранит метрики или логи, а запрашивает их у внешних систем через HTTP API и отображает в удобном виде.  
Каждый тип источника данных (Prometheus, Loki, SQL, Elasticsearch, облачные мониторы) имеет свой язык запросов и формат ответа, а Grafana приводит их к единой внутренней модели временных рядов и таблиц.

---

# Как всё это стыкуется вместе

1. **Grafana** выступает фронтендом, который через HTTP ходит в разные системы: Prometheus, Loki, SQL‑БД, Elasticsearch, Tempo, облака.  
2. Каждый datasource имеет свой язык запросов и формат ответа, но Grafana превращает их в time series и таблицы.  
3. На дашбордах можно одновременно строить панели:  
   - метрики из Prometheus;  
   - логи из Loki/Elasticsearch;  
   - трейсы из Tempo;  
   - бизнес‑показатели из PostgreSQL.  
4. Общие переменные (лейблы, namespace, сервис, окружение) позволяют связать эти панели по одному контексту и «прыгать» от графика метрик к логам и трейсам для расследования проблем.
