## Распределенная система
### — это набор независимых компьютеров или программ, которые взаимодействуют через сеть для достижения общей цели и представляются пользователю как единая система. Узлы могут быть размещены в разных географических локациях и обмениваются данными для выполнения совместных задач.

## Основные свойства распределенных систем

Распределенные системы обладают несколькими ключевыми характеристиками:

- **Совместное использование ресурсов** — оборудование, программное обеспечение и данные могут разделяться между узлами системы
- **Параллельная обработка** — одну функцию могут одновременно обрабатывать несколько машин, что повышает производительность
- **Масштабируемость** — система может горизонтально расширяться путем добавления новых узлов для обработки растущих нагрузок
- **Отказоустойчивость** — система продолжает работать даже при сбое отдельных узлов, так как другие узлы берут на себя их задачи
- **Прозрачность** — узлы могут обращаться друг к другу и обмениваться данными, а пользователи получают доступ к ресурсам без необходимости знать о внутренней сложности системы
- **Неоднородность** — узлы могут иметь различное оборудование, программное обеспечение и сетевые конфигурации

## CAP теорема

CAP теорема — это фундаментальный принцип проектирования распределенных систем, утверждающий, что невозможно одновременно гарантировать все три свойства:

- **Consistency (Согласованность)** — все узлы видят одинаковые данные в один момент времени
- **Availability (Доступность)** — каждый запрос получает ответ о успехе или неудаче операции
- **Partition tolerance (Устойчивость к разделению)** — система продолжает работать даже при потере сообщений между узлами

Согласно теореме, распределенная система может гарантировать только два из трех свойств одновременно, что заставляет архитекторов делать осознанный выбор при проектировании на основе требований конкретного приложения
